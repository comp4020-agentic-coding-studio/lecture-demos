# lecture-demos

The live demo repo for the COMP4020/COMP8020 lectures. Worked in week by week
for the whole semester; this file evolves in the open along with everything
else.

## Demo beats

At the end of every demo beat, commit. Then push --- but only if you have write
access (the convenor's clone does; student clones don't, so skip the push there
rather than fighting the rejection). The push is what lets the room's `git pull`
land on the exact state the projector is showing.

When a beat ends at a pause, also write the pause prompt to `PAUSE.md` at the
repo root (overwriting the previous one) before committing, so it travels with
the push.

Never rewrite or clean up history: the commit log is the honest record of what
actually happened, mistakes included.
